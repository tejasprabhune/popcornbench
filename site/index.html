PopcornBench
leaderboard runs repo

PopcornBench evaluates LLM agents on GPU kernel tasks derived from KernelBench. A run gives an agent a PyTorch reference module and a set of tools (compile, run for correctness, run for timing, static-check, profile, disassemble), and the agent iterates until it submits a custom CUDA kernel that matches the reference and beats it for runtime. The harness is ensemble; the world, the tools, the personas, and the layered grader (static checker, hidden reference runtime, held-out re-verification) live in this repo.

The leaderboard aggregates every published run by model. Success rate is the fraction of runs whose submission passed the agent-visible correctness check; mean speedup is the average over those successful runs of ref_runtime / kernel_runtime against the PyTorch reference. The second table breaks both down by KernelBench level (level 1 single ops, level 2 fused ops, level 3 full models, level 4 transformers, level 5 hardware translation). Held-out re-verification is on or off per cell depending on whether the run set POPCORN_HELD_OUT_SEED; cells that did not run held-out trust the agent-visible correctness check alone, which is a weaker signal than a held-out pass and worth keeping in mind when comparing rows.
